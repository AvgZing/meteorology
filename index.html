Introduction

Welcome to the new and redesigned "Learn How To Code" website! Our goal is to teach you how to code plugins for PocketMine-MP and other spoons, so that you can contribute to the large community by submitting your own plugins/addons!

Credits go to @TheKnightGamer for publicizing and sharing the website, to @remote_vase for creating the website and adding a little to the tutorials, and to @Taki21 for creating the tutorials and posting them on ImagicalMine Community forums.

Sample plugin.yml

name: PluginName
author: CoderName
api: [1.14.0, 2.0.0]
version: 1.0
main: Inactive-to-Reactive/Inactive-to-ReactiveExample.php 
# For "main" statement above, find the main class path, but exclude "src"

# Commands: If ur plugin has commands: Do this:
command:
	cmdname:
		description: cmdnameDescription
		usage: /cmdname
	cmd2name:
		description: cmd2nameDescription
		usage: /cmd2name

# Permissions 
permissions:
	cmdname.command:
		default: true
	cmd2name.command:
		default: true
Do not forget to substitute your plugin information (name, author, main path, commands, and permissions).

Volume 1: Basics and Concepts

In this first Volume, you will learn all of the items of code that are required in order for the plugin to work. You will also find the most basic functions that are optional, but they add a nice finishing (or starting, in this case) touch to the plugin!

Enable, Disable, and Load functions!

After the class statement press ENTER key and type 
public function onLoad(){
If you want a Load Message!
Press ENTER after the function line and type: 
$this->getLogger()->info("Plugin Loading");
You just created a public function and created a load message!
Here is the layout so far, and I hope common sense allows you to understand onEnable and onDisable even though we didn't give you a step by step for them:
<?php

namespace Inactive-to-Reactive;

use pocketmine\plugin\PluginBase;

class Inactive-to-ReactiveExample extends PluginBase{

          public function onLoad(){
                    $this->getLogger()->info("Plugin Loading");
          }
          public function onEnable(){
                    $this->getLogger()->info("Enabled Plugin");
          }
          public function onDisable(){
                    $this->getLogger()->info("Plugin Disabled");
          }
}
Public function onEnable is when your plugin Enables on the Logger or as we all know the CONSOLE. The plugin sends a message to the console on Enable!

Public function on Disable is when your plugin disables on the Logger or as we also know the CONSOLE. The plugin sends a message to the console on Disable!

You have learned the 3 BASIC Functions! Congratulations!


Player Events!

You know the basics! SO what now? Now we are going to learn about something that makes plugins how they are! Player Events\Events...In this lesson we will be focusing on PlayerJoinEvent and PlayerQuitEvent - Two Major Events!
To apply this in your plugin import these USE Statements! Remember "use" statements? import the following:
// Event when Player Joins
use pocketmine\event\player\PlayerJoinEvent;

// Event When Player Quits
use pocketmine\event\player\PlayerQuitEvent;

// Player "class"
use pocketmine\Player;

// Server "class"
use pocketmine\Server;

// Event Listener
use pocketmine\event\Listener;
Since we are using events now, we change the class statement a bit:
class Inactive-to-ReactiveExample extends PluginBase implements Listener{
For the onEnable() function we add this line right after public function onEnable() (Since we are USING EVENTS!!!)
public function onEnable(){
     $this->getServer()->getPluginManager()->registerEvents($this,$this); // THIS LINE
     $this->getLogger()->info("Plugin Enabled");
Do you want to broadcast a message on the server saying someone joined in a FANCY way? Then we are going to use another function!!! Here:
public function onJoin(PlayerJoinEvent $event){
   $player = $event->getPlayer();
   $name = $player->getName();
   $this->getServer()->broadcastMessage("$name Joined The Inactive-to-Reactive test Server! Awesome!");
Did you know: getPlayer() and getName() are public functions? This is how NEW Functions are made! "onJoin" is a new public function! People who know JavaScript may know this!
Congratulations on getting a Join message, but do you want it in COLOR? Import this "use" statement to your code:
// Text Format formats text and changes Color

use pocketmine\utils\TextFormat;
Now for your message:
$this->getServer()->broadcastMessage(TextFormat::GREEN."$name Joined The Inactive-to-Reactive test Server! Awesome!");
Now the Color of your message will be Green! NOTE: Instead of using getPlayer() in PlayerDeathEvent, make sure you use getEntity() or it will crash your server for everytime someone dies. A LIST of PLAYER events could be found here: ClickMeBro

Volume 2: Commands

Setting up a command

Using commands in a plugin enchants your Plugin! Commands can automatically and easily do things for you!
Insert these "use" statements if you are using commands:
// Command
use pocketmine\command\Command;

// Person who does command
use pocketmine\command\CommandSender;
To setup the command we use a public function! Layout:
public function onCommand(CommandSender $sender,Command $cmd,$label,array $args){
}
It may be long for a function, but thats how you're gonna make commands!
After Public Function onCommand():
if($cmd->getName() == "test"){ //In this case, we will make the command "/test"
     $sender->sendMessage("This is a Test");
Hmm. Now you are using "if" statements! Very Important in PHP! In this case: Explanation: If the Command Name is /test (in this case), Send a message to the player who executed this Command!
PREVIEW: Lets see what we got so far...
<?php

namespace Inactive-to-Reactive;

use pocketmine\plugin\PluginBase;
use pocketmine\event\Listener;
use pocketmine\event\player\PlayerJoinEvent;
use pocketmine\utils\TextFormat;
use pocketmine\command\Command;
use pocketmine\command\CommandSender;

class Inactive-to-ReactiveExample extends PluginBase implements Listener{
     public function onEnable(){
          $this->getServer()->getPluginManager()->registerEvents($this,$this);
          $this->getLogger()->info("Plugin Enabled");
     }
     public function onJoin(PlayerJoinEvent $event){
          $player = $event->getPlayer();
          $name = $player->getName();
          $this->getServer()->broadcastMessage(TextFormat::GREEN."$name Joined The Inactive-to-Reactive test Server! Awesome!");
     }
     public function onCommand(CommandSender $sender, Command $cmd, $label, array $args){
          if($cmd->getName() == "test"){
               $sender->sendMessage("This Is A Test!");
          }
          return true;
     }
}
The "return true" statement goes AFTER the if brackets "{" and "}" So when some one does /test they get a message "This is a test!" You want to do something better with your command? Maybe this will help:
On command the player will receive an item!
if($cmd->getName() == "test"){
     $sender->getInventory()->addItem(Item::get(364,0,2)); //364 = Item; Steak
     $sender->sendMessage(TextFormat::BOLD."OMG YOU GOT 4 STEAK!!!");
}
return true;
You got 4 steak using a Command! This is one of the great powers of commands! WAIT!!! Somethings not right! If the console was the CommandSender What will happen? How do we prevent the Console? To prevent the situation above we are going to use an if statement including "instanceof" 
if($cmd->getName() == "test"){
     if(!$sender instanceof Player){ // Basically this checks if the Command Sender is NOT a player
          $sender->sendMessage("This Command Only Works for players!"); // For Console Command Sender
     }else{ //if not a CONSOLE
          $sender->getInventory()->addItem(Item::get(364,0,4));
          $sender->sendMessage("You Got 4 Steak!");
     }
}
return true;
This will prevent the console from performing this command!
